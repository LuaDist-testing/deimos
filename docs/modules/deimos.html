Module deimos

This is the Deimos strong-typing library.

Deimos is a class system that uses metaprogramming to enforce strong type constraints. The intended use is for validating configuration files, untrusted user input, business objects that backend to databases, web forms, and the like. Odds are it can be easily abused into a true class system like LOOP or Coat; however this isn't recommend as performance was never a serious consideration.

Functions

x.any (default)
A type permitted to be anything, including nil. It imposes no constraints and thus acts exactly like any other lua variable, although you can set a default value. The default value is nil unless specified.

Parameters:

  • default: The default value when this field is initalized.
x.string (default, nullable, pattern, max_length)
A type permitted to be a string. It can be nullable or not; have a maximum length; or be forced to pass a pattern match.

Parameters:

  • default: The default value of this field. Normally nil if nullable, or the null string "" if not.
  • nullable: If true, this field can be set to nil. Default false.
  • pattern: A regular expression pattern that a set value must match in order for the set to suceed. Default, not pattern is enforced.
  • max_length: The maximum length, as determined by string.len, that this field can be set to.
x.number (default, nullable, maximum, minimum)
A type containing a number, either integer or floating point.

Parameters:

  • default: The default value of this field. Normally nil if nullable, or zero if not.
  • nullable: If true, this field can be set to nil. Default false.
  • maximum: The maximum value this field can be set to.
  • minimum: The minimum value this field can be set to.
x.bool (default)
A boolean type. There isn't much to say or do with these, considering that they by definition only have two states. In addition, Lua's handling of them is pretty good already. This exists purely to allow definition in other larger types.

Parameters:

  • default: The default value, true or false. Defaults to false.
x.class (data)
Table type 'class'. A 'class' table acts like a table with a fixed set of key/value pairs that are valid; and each parameter may have a forced type as well. Simply add functions and it behaves much like an object, except it lacks inheritance. Actual methods for the type are contained in deimos/class.lua.

Parameters:

  • data: Key/value pairs that correspond to a field's name, and a type as provided by other Deimos constructor functions.
x.array (permitted)
Table type 'array'. An 'array' table stays integer-indexed and enforces type constraints on the values within. Actual methods for the type are contained in deimos/array.lua.

Parameters:

  • permitted: The permitted data type that can be contained within the array, as provided by a deimos constructor function.
x.hash (key, value)
Table type 'hash'. An 'hash' type uses key-value pairs, like a Perl hashtable or PHP associative array. It enforces constraints on both the keys and the values within. Actual methods for the type are contained in deimos/hash.lua.

Parameters:

  • key: The data type permitted to be the key of a hash, as provided by a deimos constructor function.
  • value: The permitted data type that can be contained within the hash, as provided by a deimos constructor function.
